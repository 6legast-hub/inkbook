# INK BOOK — установка с нуля

Пошаговая инструкция. Актуальна на июнь 2026. Делается один раз, ~30–40 минут. Программировать не нужно — только регистрировать аккаунты и копировать значения. Делай по порядку, не пропуская шаги.

Что получится: сайт вида `https://inkbook.vercel.app`, куда клиенты приходят записываться, и кабинет мастера на `/login`. Всё бесплатно.

> **Совет:** держи открытыми заметки и сразу выписывай туда все ключи и значения (помечаю их 🔑). В конце перенесём их одним списком на Vercel.
>
> Интерфейсы сайтов иногда меняют надписи и расположение кнопок. Если точное название не совпало — ищи кнопку по смыслу рядом; шаги по сути остаются те же.

---

## Шаг 0. Регистрация аккаунтов (5 минут)

Зарегистрируйся (удобно «Continue with Google») на трёх сайтах:

1. **GitHub** — github.com (хранилище кода)
2. **Supabase** — supabase.com (база данных)
3. **Vercel** — vercel.com (хостинг сайта)

Telegram и Resend подключим по ходу.

---

## Шаг 1. Загрузить код на GitHub (5 минут)

1. На github.com нажми **New** (или «+» вверху справа → New repository).
2. Имя: `inkbook`. Тип — **Private** или Public, без разницы. **Create repository**.
3. На странице репозитория нажми ссылку **uploading an existing file** (в блоке «Quick setup»).
4. Распакуй `inkbook.zip` на компьютере и перетащи **всё содержимое папки** в окно загрузки (файлы и папки целиком, не саму папку-обёртку).
5. Внизу — **Commit changes**.

> Репозиторий должен быть в **твоём личном** аккаунте GitHub, а не в организации — бесплатный Vercel не подключает репозитории организаций.

---

## Шаг 2. База данных Supabase (10 минут)

### 2.1. Создать проект
1. На supabase.com → **New project**.
2. Имя `inkbook`, придумай **Database Password** (запиши 🔑), регион поближе (например Frankfurt). **Create new project**, подожди ~2 минуты.

### 2.2. Выполнить три скрипта базы
1. Слева открой **SQL Editor** (иконка вида `>_`).
2. **Кнопку «New query» искать не нужно** — посередине уже открыто пустое поле редактора с курсором. Просто кликни в него.
3. Открой на компьютере файл `supabase/migrations/001_init_schema.sql`, скопируй **всё** содержимое, вставь в поле и нажми **Run** (зелёная кнопка справа снизу, либо Ctrl+Enter / Cmd+Enter). Дождись зелёного **Success**.
4. Очисти поле (выдели всё → удали) **или** нажми «**+**» сверху, чтобы открыть новую вкладку. Вставь `002_storage.sql` → **Run**.
5. Так же выполни `003_notifications.sql` → **Run**.

> Если выскочит ошибка «… already exists» — значит скрипт уже выполнялся ранее. Тогда напиши мне текст ошибки, подскажу.

### 2.3. Скопировать URL и ключи
Нажми кнопку **Connect** вверху страницы проекта — в открывшемся окне есть **Project URL** и ключи. Либо: слева **Settings** (шестерёнка) → **API Keys**.

Выпиши три значения 🔑:

| Что копируем | Куда пойдёт (имя переменной) |
|---|---|
| **Project URL** (вида `https://xxxx.supabase.co`) | `NEXT_PUBLIC_SUPABASE_URL` |
| Ключ для клиента | `NEXT_PUBLIC_SUPABASE_ANON_KEY` |
| Секретный серверный ключ | `SUPABASE_SERVICE_ROLE_KEY` |

Про ключи (Supabase сейчас в процессе смены формата — у тебя может быть один из вариантов):

- **Если видишь вкладку «Legacy API keys»** — бери оттуда: `anon` (public) → в `NEXT_PUBLIC_SUPABASE_ANON_KEY`, а `service_role` (нажми «Reveal») → в `SUPABASE_SERVICE_ROLE_KEY`. Это самый надёжный вариант для этого проекта.
- **Если ключей `anon`/`service_role` нет, а есть новые** `sb_publishable_…` и `sb_secret_…` — тогда `publishable` → в `NEXT_PUBLIC_SUPABASE_ANON_KEY`, а `secret` → в `SUPABASE_SERVICE_ROLE_KEY`. (Имена наших переменных не меняем, меняется только что в них кладём.)

> `service_role` / `secret` — **секрет**. Никому не показывай, не выкладывай в публичный код.

### 2.4. Создать аккаунт мастера
1. Слева **Authentication** → **Users** → **Add user** → **Create new user**.
2. Email и пароль мастера (это и есть логин в кабинет 🔑). Включи **Auto Confirm User**, если есть галочка. **Create user**.
3. Нажми на созданного пользователя → скопируй **User UID** (длинная строка) 🔑.

### 2.5. Заполнить данные мастера
1. **SQL Editor** → пустое поле (или новая вкладка «+»).
2. Открой `supabase/seed_master.sql`, скопируй, вставь.
3. **Замени везде** `<MASTER_USER_ID>` на User UID из 2.4 (удобно через «найти и заменить» в текстовом редакторе перед вставкой). При желании поправь имя, адрес, инстаграм, часы и услуги — но это необязательно, всё меняется потом в кабинете.
4. **Run** → **Success**.

База готова.

## Шаг 3. Telegram-бот для уведомлений мастеру (5 минут)

### 3.1. Создать бота
1. В Telegram открой **@BotFather** → отправь `/newbot`.
2. Задай имя и username (должен оканчиваться на `bot`).
3. Скопируй присланный **токен** вида `1234:ABC...` 🔑 → это `TELEGRAM_BOT_TOKEN`.

### 3.2. Узнать chat_id мастера
1. Мастер (кто будет получать заявки) находит своего бота по username, жмёт **Start** и пишет любое сообщение.
2. В браузере открой ссылку, подставив токен:
   `https://api.telegram.org/bot<ТОКЕН>/getUpdates`
3. В ответе найди `"chat":{"id":123456789` — это число 🔑 → `TELEGRAM_MASTER_CHAT_ID`.
   *(Пусто? Значит мастер ещё не написал боту — напишите и обновите страницу.)*

## Шаг 4. Email клиентам — опционально (можно пропустить)

Нужно только для автонапоминаний клиенту на почту. **Уведомления мастеру в Telegram работают и без этого шага.**

1. Регистрация на **resend.com**.
2. **API Keys** → **Create API Key** → скопируй 🔑 → `RESEND_API_KEY`.
3. Нюанс бесплатного тарифа: без подтверждённого домена письма уходят **только на твою собственную почту** (для теста). Чтобы писать реальным клиентам → Resend → **Domains** → добавь и подтверди домен, потом впиши его в `RESEND_FROM`, например `INK BOOK <hello@твойдомен.ru>`. Домен можно добавить позже.

Пропускаешь шаг → просто оставь `RESEND_API_KEY` пустым.

---

## Шаг 5. Секрет для напоминаний

Придумай длинную случайную строку (например 30 случайных букв и цифр) 🔑 → это `CRON_SECRET`.

---

## Шаг 6. Публикация на Vercel (5 минут)

1. На vercel.com → **Add New… → Project**.
2. Найди репозиторий `inkbook` → **Import**. (Не виден? Нажми «Adjust GitHub App Permissions» / «Configure GitHub App» и дай доступ к репозиторию.)
3. Раскрой раздел **Environment Variables** и добавь все пары из заметок 🔑:

   | Name | Value |
   |---|---|
   | `NEXT_PUBLIC_SUPABASE_URL` | Project URL (2.3) |
   | `NEXT_PUBLIC_SUPABASE_ANON_KEY` | клиентский ключ (2.3) |
   | `SUPABASE_SERVICE_ROLE_KEY` | секретный ключ (2.3) |
   | `TELEGRAM_BOT_TOKEN` | токен (3.1) |
   | `TELEGRAM_MASTER_CHAT_ID` | число (3.2) |
   | `CRON_SECRET` | строка (5) |
   | `RESEND_API_KEY` | из (4) или пусто |
   | `RESEND_FROM` | `INK BOOK <onboarding@resend.dev>` |

   `NEXT_PUBLIC_APP_URL` пока пропусти — добавим после деплоя.
4. **Deploy**. Через 1–2 минуты появится адрес, например `https://inkbook-xxxx.vercel.app` — скопируй.
5. **Settings → Environment Variables** → добавь `NEXT_PUBLIC_APP_URL` = этот адрес (без слэша в конце).
6. **Deployments** → у верхнего справа «**…**» → **Redeploy**, чтобы адрес подхватился в уведомлениях.

Проверь: открой адрес — увидишь лендинг. Зайди на `/login` под email/паролем мастера (2.4) — попадёшь в кабинет.

---

## Шаг 7. Точные напоминания клиентам (5 минут)

Бесплатный Vercel запускает задачи только **раз в сутки** — для «за 1 час» мало. Подключим бесплатный внешний планировщик (он же будит базу, чтобы не «уснула»).

1. Регистрация на **cron-job.org**.
2. **Create cronjob**.
3. **Title:** `inkbook reminders`.
4. **URL:** `https://ТВОЙ-АДРЕС.vercel.app/api/cron/reminders?secret=ТВОЙ_CRON_SECRET`
   (подставь адрес сайта и строку `CRON_SECRET` из шага 5).
5. **Schedule:** каждые 30 минут.
6. **Create**. Нажми у задачи **Run now** — статус должен быть зелёным (200).

## Готово 🎉

- **Клиентам** — даёшь адрес сайта (в шапку Instagram, QR-код).
- **Мастеру** — вход `/login` (email и пароль из 2.4). С телефона: открыть `/login`, войти, «Добавить на главный экран».

### Что мастер делает в кабинете
- **Сегодня** — записи на сегодня/завтра, подтверждение заявки одной кнопкой.
- **Заявки** — все заявки и фильтры; карточка с контактами и референсами; подтвердить / перенести / отменить / «завершено» / «не пришёл».
- **Расписание** — рабочие часы и выходные/отпуск.
- **Клиенты** — история по каждому.

Новая заявка → мастеру приходит сообщение в Telegram. Подтвердил → клиенту (если оставил email) уходит письмо. За 24 ч и 1 ч до сеанса клиенту приходит напоминание.

## Если что-то не работает

- **SQL Editor: «нет New query»** → это нормально, поле уже открыто, просто вставляй в него и жми Run.
- **Ошибка про ключ / 401 от Supabase** → перепроверь, что клиентский ключ лежит в `NEXT_PUBLIC_SUPABASE_ANON_KEY`, а секретный — в `SUPABASE_SERVICE_ROLE_KEY` (не перепутаны местами). Проще всего взять пару `anon` + `service_role` из вкладки **Legacy API keys**.
- **Не входит в кабинет** → пароль из 2.4; проверь, что `seed_master.sql` выполнен с правильным UID.
- **Нет свободных слотов на странице записи** → в кабинете → **Расписание** задай рабочие часы.
- **Не приходят уведомления в Telegram** → проверь `TELEGRAM_BOT_TOKEN` и `TELEGRAM_MASTER_CHAT_ID`; мастер должен был хоть раз написать боту.
- **Письма клиентам не уходят** → на бесплатном Resend без своего домена письма идут только на твою почту; добавь домен.
- **Поменял переменные на Vercel** → после изменения обязательно **Redeploy**, иначе не подхватятся.
